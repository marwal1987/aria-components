Fördelar med <dialog> VS <div> + aria

Att använda <dialog> är mer semantiskt korrekt än att använda en div med aria attribut. Med dialog taggen kan skärmläsare och andra hjälpmedel uppfatta att det är ett modalt fönster utan att man behöver lägga till aria attribut. Om vi går på vad W3C själva säger så ska vi som utvecklare välja det mest semantiskt korrekta alternativiet. "Ingen aria är bättre än dålig aria" är ett uttryck för det. När man använder aria skapar man samtidigt ett löfte om att den specifika rollen man tilldelar sitt element ska/förväntas innehålla de interaktiva egenskaper som motsvarande semantiskt korrekta element har. Det kan skapa problem och missförtånd om man inte ser till att få allt rätt. Ur en utvecklares "tillgänglighets perspektiv" (om man kan kalla det för det), tycker jag att det blir tydligare kod att använda dialog-taggen. Sen finns det flera inbyggda egenskaper som gör att JavaScript koden innehåller färre rader. I den här uppgiften exakt hälften faktiskt. showModal() closeModal(), och "autofocus" i HTML samt att man inte behöver lägga till en eventlistener, keydown på stäng-knappen för att kunna stänga med ESC.
